Understanding the DOM

1. What is the DOM?

The DOM (Document Object Model) is a programming interface for web documents. It represents the structure of the document as a tree of objects that can be manipulated using JavaScript.




Sample Page

    
Hello World!

The above structure is represented as a tree:

  • The document object is the root node.
  • Nodes represent elements, attributes, text, and more.
2. Types of Nodes

There are several types of nodes in the DOM:

  • Element Nodes: Represent HTML elements (e.g., <div>, <p>).
  • Text Nodes: Represent the text inside an element.
  • Attribute Nodes: Represent attributes of elements.
  • Comment Nodes: Represent comments in the HTML.
// Example:
const elementNode = document.querySelector("h1"); // Element Node
const textNode = elementNode.firstChild;          // Text Node
console.log(textNode.nodeValue);                  // Logs the text inside the
3. Children of a Node

In the DOM, nodes can have children. Children can be:

  • Child Nodes: Immediate children of a node.
  • Descendant Nodes: All nodes that are nested within a node.
// Example:
let parent = document.querySelector("body");
console.log(parent.childNodes); // List of child nodes
console.log(parent.children);   // List of child elements
4. First and Last Child

Access the first and last child of a node using:

  • elem.firstChild: The first child node.
  • elem.lastChild: The last child node.

Note: Child nodes are collections, not arrays. Use Array.from(collection) to convert them into arrays.

// Example:
let parent = document.querySelector("body");
console.log(parent.firstChild);  // First child node
console.log(parent.lastChild);   // Last child node
5. DOM Properties

The DOM is live, meaning changes to the structure are immediately reflected.

  • Some properties like childNodes are read-only.
  • Collections (like childNodes) can be iterated using a for...of loop.
// Example:
let children = document.body.childNodes;
for (let node of children) {
    console.log(node);
}
6. Sibling and Parent Navigation
  • elem.nextSibling: The next sibling node.
  • elem.previousSibling: The previous sibling node.
  • elem.parentNode: The parent node.
// Example:
let element = document.querySelector("h1");
console.log(element.parentNode);     // Parent node
console.log(element.nextSibling);    // Next sibling
console.log(element.previousSibling); // Previous sibling
7. Searching the DOM

Common methods to search the DOM:

  • getElementById(): Search by ID.
  • getElementsByTagName(): Search by tag name.
  • getElementsByClassName(): Search by class name.
  • querySelector(): Search using a CSS selector.
  • querySelectorAll(): Search all matching elements.
// Example:
let byId = document.getElementById("header");
let byTag = document.getElementsByTagName("p");
let byClass = document.getElementsByClassName("content");
let bySelector = document.querySelector(".content");
let byAll = document.querySelectorAll("div");
8. Matches, Closest, and Contains
  • matches(): Checks if an element matches a selector.
  • closest(): Finds the nearest ancestor matching the selector.
  • contains(): Checks if an element is contained within another element.
// Example:
let element = document.querySelector("h1");
console.log(element.matches("h1"));          // true
console.log(element.closest("body"));        //  element
console.log(document.body.contains(element)); // true
