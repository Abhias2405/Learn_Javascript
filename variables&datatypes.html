JavaScript Variables

1. What is a Variable?

A variable is a container that stores a value. In JavaScript, you can declare a variable using var, let, or const. For example: 


var a = 4; // Here, 'a' is the identifier, and '4' is the literal.

Variables allow you to store and manipulate data in your programs. They can hold different types of data, such as numbers, strings, or objects.

2. Declaring Variables: var, let, and const

Variables in JavaScript can be declared using three keywords:

  • var: Declares a variable globally or locally (if inside a function). It can be redefined and updated.
  • let: Declares a block-scoped variable, meaning it is limited to the block, statement, or expression in which it's used. It can be updated but not redefined.
  • const: Declares a block-scoped constant. Its value cannot be changed or redefined.

Examples:


// var example:
var x = 10;
console.log(x); // Output: 10
x = 20; 
console.log(x); // Output: 20 (var can be updated)

// let example:
let y = 15;
console.log(y); // Output: 15
y = 25; 
console.log(y); // Output: 25 (let can be updated)

// const example:
const z = 30;
console.log(z); // Output: 30
// z = 40; // Error: Assignment to constant variable.

Use let and const in modern JavaScript development to avoid unexpected behaviors caused by var's global scope.

3. Variable Naming Rules

When naming variables in JavaScript, follow these rules:

  • Variable names can contain letters, digits, underscores (_), and dollar signs ($).
  • Variable names must begin with a letter, underscore (_), or dollar sign ($), but cannot begin with a digit.
  • JavaScript variable names are case-sensitive (e.g., myVariable and myvariable are different).
  • Reserved words (like let, class, and return) cannot be used as variable names.
  • Use meaningful names that describe the purpose of the variable (e.g., use totalAmount instead of x).

Examples:


// Valid variable names:
let _counter = 0;
let $price = 100;
let userName = "John";

// Invalid variable names:
// let 1value = 5; // Error: Variable name cannot start with a digit.
// let return = 10; // Error: Cannot use reserved keywords.
4. Data Types

JavaScript supports several data types, which can be broadly categorized into:

  • Primitive Data Types: Represent single values.
    • Number: Represents numeric values (e.g., 42, 3.14).
    • String: Represents textual data (e.g., "Hello", 'World').
    • Boolean: Represents logical values (true or false).
    • Undefined: Indicates a variable that has been declared but not initialized.
    • Null: Represents an empty or non-existent value.
    • Symbol: Represents unique identifiers (introduced in ES6).
    • BigInt: Represents large integers beyond the safe Number range (introduced in ES11).
  • Non-Primitive (Reference) Data Types: Objects, arrays, and functions are included in this category.

Examples:


// Primitive Data Types:
let age = 25; // Number
let name = "Alice"; // String
let isLoggedIn = true; // Boolean
let address; // Undefined
let nothing = null; // Null
let uniqueId = Symbol("id"); // Symbol
let largeNumber = 123456789012345678901234567890n; // BigInt

// Non-Primitive Data Type (Object):
let user = { firstName: "John", lastName: "Doe" };

// Array:
let numbers = [1, 2, 3, 4];

// Function:
function greet() {
    return "Hello!";
}
5. Complex Data Types

JavaScript also supports complex data types, such as objects, arrays, and combinations of the two. These types allow for the creation of more structured and dynamic data structures.

1. Array of Objects

You can store multiple objects inside an array, making it easier to manage related pieces of data.


// Example: Array of Objects
const users = [
    {
        name: "Aryan",
        age: 20
    },
    {
        name: "Priya",
        age: 22
    }
];

// Accessing an object in the array
const user1 = users[0]; // First object in the array
console.log(user1); // Output: { name: "Aryan", age: 20 }

// Accessing a property of an object in the array
const user1Age = users[0].age;
console.log(user1Age); // Output: 20

2. Nested Objects

Objects can have other objects as properties, allowing for nested structures.


// Example: Nested Object
const user1 = {
    name: "Rohan",
    age: 25,
    address: {
        city: "Mumbai",
        country: "India",
        street: "123 Main Street"
    }
};

// Accessing a nested property
const city = user1.address.city;
console.log(city); // Output: Mumbai

These complex data types are extremely useful for creating and managing structured data, especially when working with APIs or databases.
